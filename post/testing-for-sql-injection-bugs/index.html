Mindmap for SQL Injection.

Jul 5, 2021 · 3 min read

What is the Objective of this post?

Hello in this brief post i want to give a debrief of sql injection and how can a penetration tester check for sql injection vulneratbilities.

What is SQL Injection?

SQL injection is a process in which attacker/tester can manipulate, the way in which SQL query gets interpreted or processed by the underlying application. By doing so the attacker can get access to data to which he is not authorized to view/edit/insert/delete.

How to check for SQL injection vulnerabilities?

SQL Injection can be found by doing systamatic testing on application entry points as shown below.

STEP 1

By inserting quote on the application entry point (ie., a input parameter) and checking how the system responds to the input we can find whether the system is vulnerable to sql injection or not.
  • If there is a visible difference like,
    • 1 The system may throw an error.
    • 2 There might be a rendering change in the output, where the vulnerable variable is reflected
  • If there is no visible change as said previosuly.
    • 1. Try to inject an error in to the application like divided by zero and see how it is handled by the system
    • 2. Try to use sleep statement and find if there is a delay in the response of the system.
    • Note: The above two techniques need the knowledge of the type of database. As we are unable to know whether the system is vulnerable or not there is no chance of knowing the type of database. So, we can do trail and error only.

STEP 2

  • Once the system is found out to be vulnerable we should deploy different techniques to find the type of database employed. This can be done as shown below
    • If the injection is taking place and there is visible change in the output then we can use union technique

    UNION TECHNIQUE

    • This technique can be used to display different columns of a table by concatinating them with regular output. 
      # The following code helps in determing the type of database
Eg:
    # If it is a mysql database and the application logic is returning different products using the below statement 
    # For ***Oracle***
    select productname,price from products where producttype='Mobiles' union SELECT banner,null FROM v$version
    # For ***MYSQL/MSSQL***
    select productname,price from products where producttype='Mobiles' union SELECT @@version,null
    # For ***PostgreSQL***
    select productname,price from products where producttype='Mobiles' union  	SELECT version(),null
      • Simillary user can check different tables avaialable in the schema along with their columns name by unioning them with the regualar output.
  • If we found that the system is vulnerable but we cannot visible see the difference like slow response or any application errors. Then we should check for Out of band Application Security Testing or OAST is short

    Outofband Application Security Testing

    • By using tools like burp collabrator we can make database server to perform a dns lookup to external domain where the burp collabarator replies with the ip and also intimates us about the name lookup.

Places where SQL Injection takes place

  • Inside UPDATE statements, within the updated values or in the WHERE clause.
  • Inside INSERT statements, within the inserted values.
  • In SELECT statements, within the table or column name.
  • In SELECT statements, within the ORDER BY clause.
